# v0.5.22 Feature Design: CLI-Based OAuth Providers

**Version**: v0.5.22
**Release Date**: 2026-03-08
**Status**: Completed

---

## Overview

为 KodaX 添加两个新的 CLI-based OAuth Provider，从本地已安装的 CLI 工具提取凭证，无需用户提供 API Key。

### 016: CLI-Based OAuth Providers

- **Category**: New
- **Status**: Completed
- **Priority**: High
- **Design**: This document

---

## Goals

1. ✅ **Gemini CLI Provider** - 从 `~/.gemini/` 提取 OAuth 凭证
2. ✅ **Codex CLI Provider** - 从 `~/.codex/` 提取 OAuth 凭证
3. ✅ **自动 Token 刷新** - 过期时自动刷新 access token
4. ✅ **SSE 流式响应** - 支持 Server-Sent Events 流式输出
5. ✅ **Thinking 支持** - Codex 支持 reasoning 模式

---

## Architecture

### Security Model

采用 **openclaw** 的合规方式：从已安装的 CLI 工具提取凭证。

**关键原则**:
- 不硬编码 Client ID/Secret
- 从 CLI 缓存文件读取已授权的凭证
- 显示风险提示（非官方集成）
- 支持环境变量覆盖

**对比 pi-mono 方案**:
- pi-mono 要求用户提供 Google Cloud OAuth Client ID/Secret
- 用户明确拒绝此方案（安全风险）
- openclaw 方案更安全：读取已授权的 CLI 凭证

## Implementation

### Gemini CLI Provider

**文件**: `packages/ai/src/providers/gemini-cli.ts`

**凭证来源**:
1. 环境变量优先：`GEMINI_CLI_ACCESS_TOKEN`, `GEMINI_CLI_REFRESH_TOKEN`
2. CLI 缓存：`~/.gemini/oauth_creds.json` + `~/.gemini/settings.json`

**API 端点**:
```
POST https://cloudcode-pa.googleapis.com/v1internal:streamGenerateContent?alt=sse
```

**认证头**:
```typescript
{
  'Authorization': `Bearer ${accessToken}`,
  'User-Agent': 'KodaX-GeminiCLI/1.0',
  'X-Goog-Api-Client': 'genai-code-assist/0.0.0',
  'Content-Type': 'application/json',
}
```

**Project ID 发现**:
1. 从 `settings.json` 读取 `project_id`
2. 调用 Google API `loadCodeAssist` + `onboardUser` 自动发现

**Token 刷新**:
```typescript
POST https://oauth2.googleapis.com/token
{
  grant_type: 'refresh_token',
  refresh_token: string,
  client_id: string,
  client_secret: string,
}
```

**Thinking 支持**:
```typescript
if (thinking) {
  body.generationConfig = {
    ...body.generationConfig,
    thinkingConfig: { thinkingBudget: 24576 },
  };
}
```

### Codex CLI Provider

**文件**: `packages/ai/src/providers/codex-cli.ts`

**凭证来源**:
1. 环境变量优先：`CODEX_CLI_ACCESS_TOKEN`, `CODEX_CLI_REFRESH_TOKEN`
2. CLI 缓存：`~/.codex/auth.json`

**API 端点**:
```
POST https://chatgpt.com/backend-api/codex/responses
```

**认证头**:
```typescript
{
  'Authorization': `Bearer ${accessToken}`,
  'Content-Type': 'application/json',
  'Accept': 'text/event-stream',
  'ChatGPT-Account-Id': accountId, // 从 JWT 解码
}
```

**Account ID 提取**:
```typescript
function decodeJwtAccountId(token: string): string | null {
  const payload = JSON.parse(atob(token.split('.')[1]));
  return payload?.['https://api.openai.com/auth']?.chatgpt_account_id;
}
```

**Token 刷新**:
```typescript
POST https://auth.openai.com/oauth/token
{
  grant_type: 'refresh_token',
  refresh_token: string,
  client_id: 'app_EMoamEEZ73f0CkXaXp7hrann',
}
```

**Reasoning 支持**:
```typescript
if (thinking) {
  body.reasoning = { effort: 'medium', summary: 'auto' };
}
```

## Usage

### Gemini CLI

```bash
# 1. 安装并登录 Gemini CLI
npm install -g @anthropic-ai/gemini-cli
gemini auth login

# 2. 使用 KodaX
kodax --provider gemini-cli "Hello"
```

### Codex CLI

```bash
# 1. 安装并登录 Codex CLI
npm install -g @openai/codex
codex --full-setup

# 2. 使用 KodaX
kodax --provider codex-cli "Hello"
```
