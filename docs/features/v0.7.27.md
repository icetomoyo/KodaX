## 总览

v0.7.27 是 SDK 基底段的**尾声**。两件事都是结构性 hygiene 工作，风险域互不相交（前缀清理在 coding/core 核心代码，协议抽取在 repo-intelligence），可以同版本完成：

- **FEATURE_086**：借用户量少的窗口**一次性**删除 `KodaX*` 前缀（不走长期 deprecated 路径）+ 清除 legacy 代码
- **FEATURE_091**：把 `premium-contract.ts` 抽成独立 npm 包，解决当前公仓-私仓手工 vendor 的跨仓脆弱性

## FEATURE_086: KodaX Prefix Cleanup and Legacy Purge

**Status**: Planned  
**Priority**: High  
**Category**: Core

### 背景

- 当前 public API 有 100+ `KodaX*` 前缀类型（`KodaXMessage`、`KodaXSessionCompactionEntry`、`KodaXAmaControllerDecision`……）
- 品牌前缀让通用概念（Message / Session / Tool）带上"vendor 专属"暗示，劝退 SDK 消费者
- 同行对比（openai-agents-python / pi-mono）都是 `Agent / Tool / Session` 简名
- 用户量少的窗口是**唯一低成本**的 rename 时机；用户已确认"趁现在搞掉，不走长期 deprecated"
- GLM 指出的 legacy：`compactMessages()` (F-9)、`--team` CLI 参数（I-2）、README `/project` 流程（I-1）、归一化函数重复（F-4、F-6）

### 设计

#### 前缀处理规则

**保留前缀**（品牌信号）：
- `KodaXError` 及其子类（异常继承链）
- `KodaXClient`（门面 class）
- `KodaXOptions` / `KodaXCodingOptions`（产品级配置）
- `KodaXEvents`（回调总线）

**去前缀**（通用原语）：
- `KodaXMessage` → `Message`
- `KodaXContentBlock` → `ContentBlock`
- `KodaXToolDefinition` → `Tool`（合并 TS type）
- `KodaXSessionLineage` → 移除（替换为 `Session` + `LineageExtension`）
- `KodaXSessionCompactionEntry` → `CompactionEntry`
- `KodaXTokenUsage` → `TokenUsage`
- `KodaXAmaControllerDecision` → 内部化（不再 export）
- `KodaXReasoningMode` → `ReasoningMode`
- `KodaXProviderConfig` → `ProviderConfig`
- ... 等 ~80 个类型逐个改名

#### Legacy 代码清除

1. **`compactMessages()`** (packages/agent/src/messages.ts)：v0.7.23 已标 deprecated，本版直接 remove
2. **`--team` CLI 参数**：ADR-017 定废，本版 CLI 移除；保留 config.json 的 legacy 字段兼容一次解析（遇到打 warning + 忽略）
3. **README `/project` 命令流程**：DD §9.4 已定废，README / README_CN 更新
4. **归一化函数重复**：FEATURE_079 已把 7 个 `normalize*` 函数统一，本版删除旧的 duplicate
5. **YAML 归一化** (`sanitizeYaml` / `normalizeHooks` 在 discovery.ts + skill-loader.ts 重复，GLM F-6)：提取到 `@kodax/skills/shared/yaml.ts`
6. **Provider 配置双写** (GLM F-5)：内置 Provider 类 + `KODAX_PROVIDER_SNAPSHOTS` 合并为单一来源
7. **`getProvider()` 每次 `new` SDK 客户端** (GLM F-7)：改惰性单例
8. **旧 `OrchestrationTraceEvent` / `RepoIntelligenceTraceEvent`** (FEATURE_083 已 deprecated)：本版删除
9. **`@kodax/ai/src/cli-events/` 包职责外溢** (GLM F-8, FEATURE_082 deferred)：v0.7.24 FEATURE_082 原计划把 cli-events 搬离 `@kodax/ai`，但发现 `providers/acp-base.ts` / `providers/codex-cli.ts` / `providers/gemini-cli.ts` 三个 CLI-wrapper Provider 消费 cli-events，仅搬 cli-events 会造成循环依赖。**本版一并处理**：把三个 Provider 子类 + cli-events 一起搬到 `@kodax/coding/src/cli-bridges/`（或独立为 `@kodax/cli-bridges`，具体取决于本版评估），并把 `providers/registry.ts` 的 `gemini-cli` / `codex-cli` 静态注册改为由 coding 侧 side-effect 注册的 extensibility 机制。`@kodax/ai` 回到纯 LLM 抽象层定位（ADR-001）

#### Codemod + 全测试绿

- 提供一个 npm script `npm run codemod:rename-to-v0.7.27` 自动跑 rename（基于 ts-morph）
- 对 KodaX 内部 code: codemod 自动改
- 对外部消费者：提供迁移文档 + codemod 脚本发布到 npm `@kodax-author/codemod`
- Rollout 流程：跑 codemod → npm test 必须全绿 → 人工抽查一批 auto-rename → merge

### Contract

**Breaking**：100+ 类型 rename；`compactMessages()` / `--team` / `/project` 命令流程移除。

**迁移支持**：
- Codemod 脚本一键迁移 90%+ 代码
- 迁移文档（docs/MIGRATION_0.7.27.md）列出所有 rename + 示例
- 保留一批**类型别名**（如 `KodaXSessionLineage` → `never` 或 `@deprecated` 指向新的 `Session + LineageExtension`）到 v0.8.0，**但不是长期 deprecated 路径**

### Non-Goals

- **不**修改运行时行为（纯 rename + legacy remove）
- **不**发起 v1.0.0 major bump（v0.7.x 内部清理）
- **不**改变包结构（包重组已在 FEATURE_082 完成）

## FEATURE_091: Repo-Intelligence Protocol Package Extraction

**Status**: Planned  
**Priority**: High  
**Category**: Internal

### 背景

- 公仓 KodaX 的 `packages/coding/src/repo-intelligence/premium-contract.ts` 是公仓-私仓-第三方 host 三方共享的契约
- 当前私仓 `KodaX-private/packages/repointel-core/vendor/repo-intelligence/public-bridge.js` 是**手工 vendor**（build 时从公仓复制）
- 这种 vendor 方式有三个问题：
  1. 契约变更时两边 build 时序敏感
  2. 没有 npm 语义化版本保护
  3. 第三方 host（codex/claude/opencode，见 `clients/repointel/`）只能复制引用，无法依赖一个稳定 npm 包
- 用户需求：保留 KodaX-private 的闭源价值，同时让 repointel 可被 KodaX 和第三方 coding agent 单独使用

### 设计

#### 新包：`@kodax-author/repointel-protocol`

内容（~200 行，纯类型 + 常量）：
- `REPOINTEL_CONTRACT_VERSION` 常量
- `RepoIntelligenceHost` / `RepoIntelligenceIntent` / `RepointelCommand` enum types
- `RepointelRequestPayload` / `RepointelRpcRequest` / `RepointelRpcResponse` interfaces
- `RepointelStatus` / `RepointelTraceInfo` 辅助类型

#### 消费者迁移

**消费者 1：公仓 KodaX**
- `packages/coding/src/repo-intelligence/premium-contract.ts` 内容迁到新包
- `premium-contract.ts` 改为 `export * from '@kodax-author/repointel-protocol'`（保留路径向后兼容，v0.7.40 前移除）
- `premium-client.ts` 引 `@kodax-author/repointel-protocol`

**消费者 2：私仓 KodaX-private**
- `repointel-core/vendor/repo-intelligence/public-bridge.js` 删除
- `repointel-core` / `repointel-daemon` / `repointel-cli` 三个 package.json 加 `@kodax-author/repointel-protocol` 依赖
- 导入路径从 `./vendor/repo-intelligence/*` 改为 `@kodax-author/repointel-protocol`
- `scripts/vendor-seed.mjs` 或类似 vendor 脚本移除

**消费者 3：第三方 host `clients/repointel/`**
- `install.mjs` / `doctor.mjs` / `demo.mjs` 改为可选依赖 `@kodax-author/repointel-protocol`（用于 schema 校验）
- `SKILL.md` 文档更新接入方式

#### 闭源价值保留

- 协议包只有**类型 + 常量**，不暴露私仓实现逻辑（engine、semantic index、cache、routing 仍在 `KodaX-private`）
- `premium-client.ts` 仍调用本地 daemon RPC endpoint，不暴露 daemon 的内部行为
- 第三方 host 能通过协议包理解"如何调 daemon"，但无法复现 daemon 本身

### Contract

**Public API（新 npm 包）**：`@kodax-author/repointel-protocol` 首个版本 `v1.0.0`，语义化版本保护契约变更。

**Internal**：
- 公仓 `premium-contract.ts` 变成 re-export shim
- 私仓三 package.json 增加 dep
- 第三方 `clients/repointel/scripts/*.mjs` 可选升级

### Non-Goals

- **不**改变 daemon 协议语义（只是把协议定义搬家）
- **不**合并公仓-私仓代码库
- **不**发布 daemon 的闭源内容

### Dependencies

- **前置**：无硬依赖（契约抽取独立于 SDK 基底重组）
- **后置**：无硬后置

### Testing

- 单元：协议包类型在公仓 `premium-client` 使用等价于旧 `premium-contract.ts`
- 集成：私仓 daemon 拆掉 vendor、用 npm dep 后，`repointel-daemon` + `repointel-cli` 全测试绿
- 集成：公仓 `repo-intelligence/query.ts` 对 daemon 的 RPC 调用结果与 v0.7.26 等价
- 集成：`clients/repointel/scripts/demo.mjs` 在接入新协议包后仍能跑 demo flow

### Risk

- **跨仓 coordination**：公仓先发 `@kodax-author/repointel-protocol@1.0.0`，私仓后升级依赖；需要短时间内完成两边 release 以避免跨仓不一致
- **向后兼容**：公仓保留 `premium-contract.ts` re-export 路径一段时间；私仓 vendor 移除可以稍晚

### Timeline

v0.7.27 合并 086 + 091。前缀清理 + 协议包抽取分别走各自 PR，可并行推进；本版发版前两边都要绿。
