> 中文标题：`v0.7.25 Feature 设计`
>
> Version: `v0.7.25`
>
> Status: Completed
>
> Theme: host-aware TUI substrate and transcript UX

---

## 总览

`v0.7.25` 的目标不是把 KodaX 做成另一个重表面的 terminal product。

这一版只做一件事：

把当前为了止血而存在的 TUI 妥协，收束成一套真正可持续的交互底座。

当前 KodaX 已经有可用的 REPL、多行输入、自动补全、history review，以及一些专门为 Windows terminal 问题补的 keypress 和 bracketed paste 修复；但 transcript、input、status、history browsing 之间仍然耦合得太紧。最明显的症状是：

- `review mode` 同时承担“看更多细节”“浏览历史”“暂停 live flush”三种职责
- transcript 仍然主要依赖 viewport budgeting 和 row slicing 维持稳定
- verbose / detailed output 还没有独立成一等交互语义
- Windows Terminal、VS Code terminal 这类宿主差异还没有进入 TUI 策略层

`v0.7.25` 要解决的，不是“再加几个快捷键”，而是把这些状态拆开，并给 KodaX 建立一层更薄、更稳、更聪明的 terminal substrate。

## FEATURE_051: Host-Aware Fullscreen TUI Substrate and Transcript UX

**Status**: Completed
**Close-out note (2026-04-03)**: Implemented with host-aware transcript profiles, decoupled verbosity/follow/buffering state, fullscreen-or-fallback transcript control, and stable hidden-work strip integration.
**Priority**: High
**Category**: Enhancement

### Overview

`FEATURE_051` 把 KodaX 的 TUI 明确成三层：

1. `host-aware substrate`
2. `transcript and composer interaction model`
3. `minimal but high-value surface polish`

它借鉴 ClaudeCode 做得对的部分，但不追求 surface parity。

KodaX 要学的是：

- application-owned fullscreen transcript viewport
- host-aware scrolling and fallback
- stable-height footer / status / composer layout
- transcript verbosity 与 history browsing 的职责拆分

KodaX 不该学的是：

- 过重的产品表面
- 用户可见的 worker topology
- 与极简哲学不匹配的过度功能堆叠

### Product philosophy

#### 1. 极简不是“功能少”，而是“语义清楚”

用户可以获得更强的 transcript、history、status、并行可见性，但不应该被迫理解复杂 renderer 细节或多套模式命名。

#### 2. 智能首先体现在系统自己选对交互策略

同样的 TUI 在 Windows Terminal、VS Code terminal、tmux、以及其他 xterm-compatible host 下，不应该假设一套行为都同样正确。

#### 3. Verbose 不是 review

“我要展开更多信息”与“我要浏览历史”与“系统为了稳定暂时缓冲 live updates”必须拆开。

#### 4. Fullscreen substrate 优先于 surface flourishes

如果 transcript viewport、footer 高度、mouse / wheel / focus lifecycle 还不稳，继续加更多 UI surface 只会放大闪烁和冲突。

#### 5. 为 invisible parallelism 留出表面，但不暴露拓扑

TUI 应该让用户感知系统正在推进哪些后台工作，但不该把 internal worker graph 直接搬出来。

### Goals

1. 建立 host-aware TUI substrate，让不同 terminal host 可以走不同的交互策略。
2. 把 transcript verbosity、history browsing、buffered fallback 拆成独立状态。
3. 引入 application-owned fullscreen transcript viewport，减少闪烁、跳动和 layout fights。
4. 让 `Ctrl+O` 成为正式的 transcript verbosity 开关，而不是 review fallback 的别名。
5. 给后续 hidden parallelism 和 richer footer/status 留出稳定的表面插槽。

### Scope

- terminal host classification
- fullscreen transcript viewport
- transcript verbosity toggle
- history browsing / follow-bottom state split
- buffered fallback demotion
- stable-height footer / status / composer layout
- host-specific wheel / scroll behavior
- minimal background-work surface hooks

### Non-goals

- 复制 ClaudeCode 的所有 TUI 表面能力
- 引入 voice / mobile / remote / teammate dashboard 一类额外表面
- 用 renderer rewrite 解决一切问题
- 在本 feature 中重做整个 REPL 渲染库

### Current pain to resolve

#### A. 状态耦合过重

`isReviewingHistory` 现在同时控制：

- transcript windowing
- detailed tool output
- wheel behavior
- live update pause / resume

这导致 review mode 变成一个过重的“万能开关”。

#### B. Layout 仍然偏预算驱动

当前 transcript 区域高度仍然依赖 `viewport budget` 计算，容易被：

- suggestions
- pending inputs
- status bar
- multi-line input

互相挤压。

#### C. 宿主差异没有进入策略层

KodaX 现在知道：

- 是否支持 truecolor
- 是否支持 unicode
- 是否在 Windows Terminal 或 VS Code terminal 中

但这些知识还没进入 fullscreen、wheel、buffering、fallback 的 TUI 决策。

#### D. KodaX 还没有真正的 transcript verbosity 语义

当前 detailed tool output 主要通过 review mode 打开，但用户真正想要的常常是：

- 平时保持 compact
- 按需展开到 verbose
- 不影响 live transcript 正常工作

### State model

`FEATURE_051` 需要把当前混杂的 TUI 状态拆成一份正式的 interaction state。

推荐最小状态模型：

```ts
type TranscriptVerbosity = 'compact' | 'verbose';
type TranscriptFollowMode = 'follow-bottom' | 'browsing-history';
type LiveBufferingPolicy = 'live' | 'buffered-fallback';

interface TranscriptInteractionState {
  verbosity: TranscriptVerbosity;
  followMode: TranscriptFollowMode;
  buffering: LiveBufferingPolicy;
  scrollOffset: number;
  followBottom: boolean;
  hasBufferedIncoming: boolean;
}
```

关键约束：

- `verbosity` 不再由 `browsing-history` 隐式决定
- `scrollOffset` 不应再只在 fallback review mode 中存在
- `followBottom` 是 transcript viewport 的一等状态，而不是从 `scrollOffset === 0` 推断出来的近似结果
- `hasBufferedIncoming` 只服务 UI 提示与 resume 策略，不应决定 transcript verbosity

### Host policy matrix

`FEATURE_051` 不只要做 host classification，还要定义每类 host 的默认策略。

| Host class | Example hosts | Fullscreen viewport | Mouse strategy | Wheel strategy | Browsing history policy |
|---|---|---|---|---|---|
| `native_vt` | Windows Terminal, iTerm2, kitty, ghostty | 默认开启 | 正常启用，允许更强交互 | native path | history browsing 默认不暂停 live |
| `xtermjs_host` | VS Code terminal, Cursor terminal, Windsurf terminal | 开启，但更保守 | 避免抢宿主已提供的选择/链接语义 | xterm.js tuned path | history browsing 可选 buffered fallback |
| `degraded_vt` | 能力不完整或 probe 不稳定的终端 | 默认不开 fullscreen viewport | 只保留最小 mouse 能力 | conservative path | 更早进入 buffered fallback |
| `unsupported_control_host` | 明确不兼容的 control-mode 或特殊宿主 | 禁用 fullscreen viewport | 禁用主动 mouse ownership | fallback only | 保留 append-only 或 history inspect fallback |

设计原则：

- `native_vt` 优先体验
- `xtermjs_host` 优先稳定
- `degraded_vt` 优先正确
- `unsupported_control_host` 优先不破坏宿主

### Shortcut contract

`FEATURE_051` 需要把 transcript 相关的快捷键契约正式收口。

#### Proposed shortcut semantics

- `Ctrl+O`: toggle transcript verbosity
- `PgUp`: enter browsing-history
- `Ctrl+Y`: explicit history browsing toggle
- `Alt+Z`: ergonomic backup for history browsing
- `Esc`: leave browsing-history
- `End`: jump to latest and restore follow-bottom

#### Required shortcut migration

当前 KodaX 中：

- `Ctrl+O` 仍然绑定给 `togglePermissionMode`

这与 transcript verbosity 目标冲突，必须在本 feature 中调整。

建议迁移方向：

- `Ctrl+O` 迁给 `toggleTranscriptVerbosity`
- `togglePermissionMode` 保留到 `Shift+Tab`
- 如果 `Shift+Tab` 最终不合适，再迁到其他不与 transcript 核心语义冲突的组合键

设计约束：

- transcript verbosity 必须成为比 permission-mode toggle 更基础的交互入口
- permission-mode toggle 仍需保留，但不该占用最有认知价值的 transcript 快捷键

### Component responsibility split

`FEATURE_051` 不只是加状态，还要明确模块边界。

#### `InkREPL.tsx`

负责：

- host profile 选择
- transcript interaction state 的顶层拥有
- fullscreen viewport 启用与 fallback 选择
- transcript / composer / footer / status 的布局骨架

不再负责：

- 用单一 `isReviewingHistory` 表达所有 transcript 语义

#### `MessageList.tsx`

负责：

- 接收明确的 viewport state
- 接收独立的 `verbosity`
- 接收独立的 `followMode`

不再负责：

- 将 `windowed` 直接等价成 `showDetailedTools`

#### `InputPrompt.tsx`

负责：

- 纯 composer body
- multiline 输入、history、autocomplete
- transcript browsing 状态下的 focus 契约

不应继续承担：

- 通过 placeholder 文案隐式表达整个 transcript mode

#### `StatusBar.tsx`

负责：

- 轻量状态与提示
- follow / browsing / buffered 状态可视化

不应继续承担：

- 因高度波动而反复挤压 transcript

#### `terminalCapabilities.ts`

负责：

- 升级为 terminal host profile layer 的底层信息源

建议新增的抽象：

- `detectTerminalHostProfile()`
- `getFullscreenPolicyForHost()`
- `getWheelPolicyForHost()`

### Design direction

#### 1. Introduce host classes

`FEATURE_051` 应引入最小但明确的 host classification：

- `native_vt`
- `xtermjs_host`
- `degraded_vt`
- `unsupported_control_host`

这层不只服务颜色与 Unicode，而要直接驱动：

- 是否启用 fullscreen viewport
- 是否启用 aggressive mouse integration
- wheel / scroll strategy
- 是否在 browsing history 时启用 buffered fallback

#### 2. Split transcript state into three independent axes

推荐拆成三条状态轴：

- `transcript verbosity`
  - `compact | verbose`
- `transcript follow mode`
  - `follow-bottom | browsing-history`
- `live buffering policy`
  - `live | buffered-fallback`

这样：

- `Ctrl+O` 只切 verbosity
- `PgUp` 或 wheel-up 进入 history browsing
- 只有在 host 或 runtime 判断有必要时，browsing history 才附带 buffered fallback

#### 3. Make fullscreen viewport the primary path where safe

在 `native_vt` 和稳定的 `xtermjs_host` 上，KodaX 应优先使用 application-owned transcript viewport，而不是继续让 `review mode` 承担主要历史浏览职责。

这个 viewport 至少需要：

- explicit scroll state
- explicit follow-bottom state
- jump-to-latest
- stable transcript region ownership
- buffered incoming content when necessary

#### 4. Demote review mode into a fallback path

当前的 review mode 不应该继续作为“看详细 transcript”的正式主路径。

它应收窄为：

- `history inspect fallback`
- only used when host / renderer stability is insufficient
- still available as a safety valve

#### 5. Move to stable-height footer/status/composer contracts

KodaX 需要明确规定：

- footer 高度稳定
- status 区域高度稳定
- multi-line input 的增长不能不断偷 transcript 行
- suggestions / notifications 尽量 overlay 或预留固定预算

#### 6. Keep the surface minimal but strategic

可以补的高价值 surface：

- `Ctrl+O` transcript verbosity
- lightweight status line
- jump-to-latest / unseen indicator
- small background-work pill

不应该在这一版补的 surface：

- visible agent graph
- heavy modal dashboard
- multi-pane product UI

### Deliverables

#### 051-A Host-aware terminal profile layer

新增一个正式的 terminal profile / host classification 层，负责把：

- environment signals
- terminal probes
- host quirks

收束成上层 TUI 可以消费的 `profile`。

最小输出建议：

- `hostClass`
- `supportsFullscreenViewport`
- `supportsMouseOwnership`
- `wheelPolicy`
- `scrollPolicy`
- `prefersBufferedHistoryFallback`
- `reasonIfDegraded`

#### 051-B Transcript state split

从当前单一的 review-mode 语义中拆出：

- transcript verbosity
- follow-bottom vs browsing-history
- buffered fallback policy

并给它们清晰的状态与快捷键契约。

这里的重点不是状态数量，而是职责不再重叠：

- `verbosity` 只影响显示密度
- `followMode` 只影响 scroll ownership
- `buffering` 只影响 live update 与 buffered resume

#### 051-C Fullscreen transcript viewport substrate

引入 application-owned transcript viewport：

- explicit scroll offset
- stable viewport region
- follow-bottom state
- jump-to-latest primitives

推荐 first implementation contract：

- viewport state 只由 REPL 顶层持有
- `MessageList` 接受 visible slice / scroll metadata，而不是自己推断“是否在 review mode”
- incoming streaming 内容可以与 visible slice 解耦

#### 051-D Host-specific wheel and scroll behavior

不同 host 至少支持：

- native-vt wheel strategy
- xterm.js / VS Code terminal wheel strategy
- conservative degraded fallback

这部分必须避免“一套步长打天下”。

至少要区分：

- native wheel cadence
- xterm.js browser event cadence
- degraded fallback 的低风险节流

#### 051-E `Ctrl+O` transcript verbosity

正式引入 transcript verbosity：

- compact by default
- verbose on demand
- decoupled from review/history fallback

同时要补上 collapse / expand contract：

- compact mode 可以折叠 thinking / tool output / grouped transcript details
- verbose mode 展开这些细节
- compact / verbose 不应影响 follow-bottom 逻辑

#### 051-F Stable footer / status / composer contracts

将 input、status、help、pending-inputs、suggestions 的布局契约稳定下来，减少 flicker 和 row stealing。

最重要的不是“完全不变化”，而是：

- transcript region 的高度变化可预测
- footer/status 的高度变化尽量固定或强约束
- overlay 内容不要反复推动 transcript 主体

#### 051-G Minimal hidden-work surface hooks

为后续 invisible parallelism 预留极小的表面能力：

- background work summary slot
- non-intrusive progress pill

注意：

- 这里只做 slot / hook，不做 heavy task dashboard
- 目的是给 future invisible parallelism 提供表面锚点

### Implementation slices

#### Slice 1: shortcut and state decoupling

- add `toggleTranscriptVerbosity`
- migrate `Ctrl+O` away from `togglePermissionMode`
- split `isReviewingHistory` into multiple state axes without changing renderer yet

#### Slice 2: host profile layer

- lift environment-based capability checks into host classes
- map Windows Terminal and VS Code terminal into distinct policies
- add explicit degraded and unsupported fallback decisions

#### Slice 3: fullscreen transcript ownership

- move transcript scroll ownership to REPL root
- introduce explicit `followBottom`
- stop using `windowed` as the same concept as “review mode”

#### Slice 4: stable footer/status contracts

- reduce row stealing from status, help, suggestions, and input growth
- split persistent footer rows from transient overlay rows

#### Slice 5: fallback review-path demotion

- keep history inspect fallback for unstable hosts
- remove its responsibility for transcript verbosity
- keep buffered-resume behavior only where it still adds value

### Verification matrix

`FEATURE_051` 需要明确把宿主兼容验证纳入 feature 本身。

最小验证矩阵建议：

| Host | Must verify |
|---|---|
| Windows Terminal | fullscreen viewport、Ctrl+O verbose、wheel scroll、multi-line input stability |
| VS Code terminal | xterm.js wheel behavior、Alt/selection compatibility、buffered fallback policy、footer stability |
| Plain external terminal | compact/verbose switch、history browsing、append-only fallback |

重点验证场景：

- streaming 中按 `Ctrl+O`
- streaming 中向上滚动进入 browsing-history
- browsing-history 状态恢复到 follow-bottom
- suggestions / pending inputs / status 同时出现时 transcript 不乱跳
- Windows paste / bracketed paste / multiline input 不破坏 layout

### Rollout order

1. Introduce host-aware terminal profiles.
2. Split transcript verbosity from review / history semantics.
3. Add fullscreen transcript viewport substrate for safe hosts.
4. Demote current review mode into an explicit fallback path.
5. Stabilize footer / status / composer height contracts.
6. Add lightweight hidden-work surface hooks.

### Open questions

- `togglePermissionMode` 最终迁移到哪个默认快捷键最合适
- VS Code terminal 下是否第一阶段就允许 aggressive mouse ownership
- `buffered-fallback` 是否只在 browsing-history 时生效，还是也允许在 waiting-for-input 场景中启用
- fullscreen transcript viewport 第一阶段是否需要同时支持 mouse-wheel 与 keyboard pager，还是先以 keyboard-first 稳定为主

### Relationship to other features

- `FEATURE_012` provided autocomplete and baseline TUI editing, but not host-aware transcript ownership
- `FEATURE_017` introduced pending-input visibility, which now needs a more stable footer/layout contract
- `FEATURE_023` remains the broader long-term dual-mode terminal UX feature; `051` front-loads the most urgent substrate and interaction cleanup
- `FEATURE_047` will benefit from the hidden-work surface hooks added here

### Acceptance criteria

- [ ] KodaX has an explicit host-aware terminal profile layer that can influence fullscreen, scrolling, and fallback behavior
- [ ] transcript verbosity is no longer coupled to review/history mode
- [ ] `Ctrl+O` toggles transcript verbosity independently of history browsing
- [ ] application-owned fullscreen transcript viewport is available on safe hosts
- [ ] review mode is demoted to a fallback path rather than serving as the primary detailed-transcript UX
- [ ] footer / status / composer height changes no longer cause frequent transcript jump or row stealing
- [ ] the feature improves TUI maturity without introducing a heavy visible multi-agent or multi-surface product shell

---

## Cross-feature intent for v0.7.25

1. `051` makes KodaX's terminal interaction model less dependent on stop-gap review-mode behavior.
2. `051` creates a cleaner bridge from today's Ink-based REPL to the richer substrate expected by later terminal UX work.
3. `051` gives hidden parallelism and richer transcript evidence display a place to surface gently, without violating the product's minimalism.
