## FEATURE_014: Project Mode Enhancement - AI-Driven Project Management

重新设计 `/project` 命令，采用 AI-First 和 Prompt-Driven 方法。将命令从 13 个精简到 7 个，同时提升灵活性和安全性。

**核心原则**:
1. **AI-First Design**: 所有命令都是 AI 辅助的
2. **Prompt-Driven**: 用户用自然语言描述意图
3. **Intelligent Understanding**: AI 理解 prompt 并执行操作
4. **Safety First**: 破坏性操作需要确认
5. **No Interactive Editing**: 一切通过 prompt，不需要编辑器/问答模式

### 2. 影响范围

**需要创建的文件**:

##### 3.1.2 `/project status [prompt] [--features|--progress]`
统一视图命令（合并 `list` + `progress`）。
- 默认：概览 + 统计信息
- `--features`：详细 feature 列表
- `--progress`：详细进度日志
- 带 prompt：AI 驱动的过滤和分析

**实现**:
```typescript
async function projectStatus(
  args: string[],
  storage: ProjectStorage
): Promise<void> {
  const guidance = args.filter(a => !a.startsWith('--')).join(' ');
  const showFeatures = args.includes('--features');
  const showProgress = args.includes('--progress');

  // 如果有 guidance，使用 AI 分析
  if (guidance) {
    return await intelligentStatus(guidance, storage);
  }

  // 默认：显示概览
  if (!showFeatures && !showProgress) {
    const stats = await storage.getStatistics();
    console.log(`\n📊 Project Status`);
    console.log(`  ✓ ${stats.completed}/${stats.total} completed (${stats.percentage}%)`);
    console.log(`  ⏳ ${stats.pending} pending, ${stats.skipped} skipped`);
    return;
  }

  // 显示 features 或 progress
  if (showFeatures) await listFeatures(storage);
  if (showProgress) console.log(await storage.readProgress());
}
```

```typescript
function buildFeaturePrompt(
  desc: string,
  steps?: string[],
  userPrompt?: string
): string {
  const stepsSection = steps?.length
    ? `\n\nPlanned steps:\n${steps.map((s, i) => `${i + 1}. ${s}`).join('\n')}`
    : '';

  const userSection = userPrompt
    ? `\n\nAdditional requirements:\n${userPrompt}`
    : '';

  return `Continue implementing the project. Focus on this feature:

${desc}${stepsSection}${userSection}

After completing this feature, update feature_list.json to mark it as passes: true.`;
}
```

##### 3.1.5 `/project edit [#index] <prompt>` - **核心创新**
万能编辑命令（AI 驱动）。

**单个 Feature 编辑**:
```bash
/project edit #3 "修改描述为 xxx"
/project edit #3 "标记为完成"
/project edit #3 "添加步骤：写测试"
/project edit #3 "删除"
```

**全局编辑**:
```bash
/project edit "重新按优先级排序"
/project edit "合并 #3 和 #4"
/project edit "拆分 #5，粒度太粗"
/project edit "删除所有已跳过的"
```

**合并的命令**:
- `mark <n> done` → `edit #n "标记为完成"`
- `refactor "prompt"` → `edit "prompt"`
- `split <n>` → `edit "拆分 #n"`
- `remove <n>` → `edit #n "删除"`

**实现**:
```typescript
async function projectEdit(
  args: string[],
  storage: ProjectStorage
): Promise<void> {
  // 检查第一个参数是否是 #index
  const firstArg = args[0];
  const isIndex = /^#\d+$/.test(firstArg);

  if (isIndex) {
    // 编辑单个 feature
    const index = parseInt(firstArg.slice(1), 10);
    const guidance = args.slice(1).join(' ');
    await editSingleFeature(index, guidance, storage);
  } else {
    // 编辑全局
    const guidance = args.join(' ');
    await editGlobal(guidance, storage);
  }
}

async function editSingleFeature(
  index: number,
  guidance: string,
  storage: ProjectStorage
): Promise<void> {
  const feature = await storage.getFeatureByIndex(index);
  if (!feature) {
    console.log(`Feature #${index} not found`);
    return;
  }

  // AI 理解意图
  const prompt = `Analyze user's guidance and modify this feature:

Feature: ${JSON.stringify(feature, null, 2)}

User guidance: "${guidance}"

Possible actions:
- Modify description/steps
- Mark as completed/skipped/pending
- Delete feature
- Other modifications

Output JSON with action and modified feature (if applicable):
{
  "action": "modify" | "mark_complete" | "mark_skip" | "delete" | "reset",
  "feature": { ... } // if applicable
}`;

  const result = await callAI(prompt);

  // 执行操作
  if (result.action === 'delete') {
    await storage.removeFeature(index);
    console.log(`✓ Deleted feature #${index}`);
  } else if (result.action === 'mark_complete') {
    await storage.updateFeatureStatus(index, {
      passes: true,
      completedAt: new Date().toISOString()
    });
    console.log(`✓ Marked feature #${index} as completed`);
  } else if (result.feature) {
    await storage.updateFeatureByIndex(index, result.feature);
    console.log(`✓ Updated feature #${index}`);
  }
}

async function editGlobal(
  guidance: string,
  storage: ProjectStorage
): Promise<void> {
  const features = await storage.loadFeatures();
  if (!features) return;

  // AI 理解全局意图
  const prompt = `Analyze user's guidance and modify the feature list:

Features: ${JSON.stringify(features.features, null, 2)}

User guidance: "${guidance}"

Possible actions:
- Reorder features
- Merge features
- Split features
- Delete features
- Batch operations
- Other restructuring

Output JSON with:
{
  "action": "reorder" | "merge" | "split" | "delete" | "batch" | "other",
  "features": [ ... ], // modified feature list
  "description": "what changed" // for user confirmation
}`;

  const result = await callAI(prompt);

  // 显示变更并确认
  console.log(`\n${result.description}\n`);
  console.log('Preview:');
  result.features.slice(0, 5).forEach((f, i) => {
    console.log(`  ${i}. ${f.description}`);
  });
  if (result.features.length > 5) {
    console.log(`  ... and ${result.features.length - 5} more`);
  }

  const confirmed = await confirm('\nApply changes?');
  if (confirmed) {
    await storage.saveFeatures({ features: result.features });
    console.log('✓ Changes applied');
  }
}
```

##### 3.1.6 `/project reset [--all]`
安全的重置命令，语义清晰。

**`/project init` 创建的文件**:
- `feature_list.json` (项目根目录)
- `PROGRESS.md` (项目根目录)
- `.kodax/session_plan.md` (**只删除这个文件，不删除 .kodax/ 文件夹！**)

**选项**:
- 无选项：清空 `PROGRESS.md`（保留 features）
- `--all`：删除所有 3 个 project 管理文件

**安全保证**:
- ✅ 只删除 `/project init` 创建的文件
- ✅ `.kodax/` 文件夹被保留（可能包含 settings.json, memory/ 等）
- ✅ 项目代码永远安全（src/, package.json 等）
- ✅ 删除前明确列出文件
- ✅ 需要确认

**实现**:
```typescript

async function safeDeleteProjectFiles(storage: ProjectStorage): Promise<void> {
  // 1. 只删除这 3 个特定文件
  const safeFiles = [
    storage.featuresPath,      // feature_list.json
    storage.progressPath,      // PROGRESS.md
    storage.sessionPlanPath,   // .kodax/session_plan.md (只删除这个文件！)
  ];

  // 2. 安全检查：确保文件在项目目录内（防止路径遍历攻击）
  const projectDir = storage.projectDir;
  for (const file of safeFiles) {
    const resolved = path.resolve(file);
    if (!resolved.startsWith(projectDir)) {
      throw new Error(`Safety check failed: ${file} is outside project directory`);
    }
  }

  // 3. 再次确认：.kodax/ 文件夹不会被删除
  const kodaxDir = path.dirname(storage.sessionPlanPath);
  console.log(chalk.dim(`Note: Only .kodax/session_plan.md will be deleted`));
  console.log(chalk.dim(`      .kodax/ folder and other files will be preserved`));

  // 4. 删除前再次确认
  console.log('\nFiles to delete:');
  safeFiles.forEach(f => console.log(`  ${f}`));
  const confirmed = await confirm('Proceed?');

  if (confirmed) {
    // 5. 逐个删除文件（不删除文件夹！）
    for (const file of safeFiles) {
      try {
        await fs.unlink(file);
        console.log(`✓ Deleted ${file}`);
      } catch (e) {

          console.error(`✗ Failed to delete ${file}: ${e}`);
        }
      }
    }
  }
}
```

#### 3.2 Feature Index 语法

使用 `#<number>` 引用 features：
- `#3` → 索引为 3 的 feature
- `#0` → 第一个 feature

**优势**:
- ✅ GitHub 惯例（issues #123）
- ✅ 清晰明确
- ✅ 视觉上区分命令参数
- ✅ Tab 补全应该显示 `#0 #1 #2 #3 ...`

**实现**:
```typescript
function parseFeatureIndex(arg: string): number | null {
  // 支持两种格式：
  // #3 → 3
  // 3  → 3

  if (arg.startsWith('#')) {
    const num = parseInt(arg.slice(1), 10);
    return isNaN(num) ? null : num;
  }

  const num = parseInt(arg, 10);
  return isNaN(num) ? null : num;
}
```

```typescript
// project-completer.ts
export function completeProjectCommand(
  args: string[]
): string[] {
  const subCommand = args[0]?.toLowerCase();
  const partial = args[args.length - 1] || '';

  // 补全子命令
  if (args.length === 1) {
    return [
      'init', 'status', 'next', 'auto',
      'edit', 'reset', 'analyze'
    ].filter(cmd => cmd.startsWith(partial));
  }

  // 补全 edit 命令的 #index
  if (subCommand === 'edit' && args.length === 2) {
    // 返回 #0 #1 #2 #3 ...
    return generateFeatureIndices();
  }

  // 补全 --选项
  if (partial.startsWith('--')) {
    const options = getOptionsForCommand(subCommand);
    return options.filter(opt => opt.startsWith(partial));
  }

  return [];
}

async function generateFeatureIndices(): Promise<string[]> {
  // 从 feature_list.json 读取并生成 #0 #1 #2 ...
  const storage = new ProjectStorage(process.cwd());
  const features = await storage.loadFeatures();
  if (!features) return [];

  return features.features.map((_, i) => `#${i}`);
}

function getOptionsForCommand(cmd: string): string[] {
  const optionMap: Record<string, string[]> = {
    'init': ['--append', '--overwrite'],
    'status': ['--features', '--progress'],
    'next': ['--no-confirm'],
    'auto': ['--max=', '--confirm'],
    'reset': ['--all'],
  };
  return optionMap[cmd] || [];
}
```

### 9. 设计决策

#### 9.1 为什么用 `#` 表示 Feature Index？
- ✅ GitHub 惯例（issues #123）
- ✅ 清晰明确
- ✅ 视觉上区分数字
- ❌ 考虑的替代方案：`@3`, `:3`, `f3`, `3` - 都不如 `#`

#### 9.2 为什么合并命令？
- `list` + `progress` → `status`: 都显示项目状态
- `mark` + `split` + `remove` → `edit`: 都修改 features，AI 理解意图
- 减少认知负担

#### 9.3 为什么只有 2 个 Reset 选项？
- 无选项：安全操作（清空进度）
- `--all`：破坏性操作（删除文件）
- 避免混淆（移除了 `--clean`）
- 清晰的语义

#### 9.4 为什么不删除 `.kodax/` 文件夹？
- 包含项目级配置（settings.json）
- 可能包含记忆文件、钩子
- 只有 `session_plan.md` 是 `/project init` 创建的
- 删除整个文件夹会丢失用户数据
