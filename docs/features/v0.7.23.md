# v0.7.23 Feature Designs

> 中文标题：`v0.7.23 Feature 设计`
>
> Version: `v0.7.23`
>
> Status: Planned (not yet released)
>
> Theme: Layer A data-shape lock-in — Agent / Handoff / Runner / Guardrail primitives + Compaction/Session split

---

## 总览

v0.7.23 是 KodaX 从 "coding CLI" 升级为 "可嵌入 Agent 基础设施" 的**哲学转折点**。

Plan B 把原本分在 v0.7.23 / v0.7.24 的两条相关工作合并：**FEATURE_080** 引入 4 个 Layer A 原语（Agent / Handoff / Runner / Guardrail），**FEATURE_081** 把 Session 和 Compaction 分层。两者都是"data shape 定型"工作 —— Agent 的 tools / handoffs 与 Session 的 entries 在内部需要同一批引用点一起定型，合并可以避免 touch 相同文件两次。

---

## FEATURE_080: Layer A Primitives — Agent / Handoff / Runner / Guardrail

**Status**: Planned  
**Priority**: High  
**Category**: Core

### 背景

- 外部系统想把 KodaX 当 Agent SDK 嵌入，目前只能黑盒调 `runKodaX()`；无法声明自己的 Scout / Evaluator / 业务 agent
- Scout / Generator / Evaluator 这些内部"角色"在代码里是隐式的字符串 key 和 prompt fragment，没有统一数据模型
- Reasoning 分层（FEATURE_078，v0.7.29）依赖 Agent 定义里有 `reasoning` 字段；自构建（FEATURE_087-090）依赖 Agent 是可序列化 data
- pi-mono 和 openai-agents-python 的经验：**Agent-as-data** 是可组合、可嵌入、可自构建的前提

### 设计

#### 四个原语（data shapes）

```ts
// packages/coding/src/primitives/agent.ts (FEATURE_082 v0.7.24 迁到 @kodax/core)

export interface Agent<TContext = unknown, TOutput = unknown> {
  readonly name: string
  readonly instructions: string | ((ctx: TContext) => string)
  readonly tools?: readonly Tool[]
  readonly handoffs?: readonly Handoff[]
  readonly reasoning?: AgentReasoningProfile  // FEATURE_078 v0.7.29 实装
  readonly guardrails?: readonly Guardrail[]  // FEATURE_085 v0.7.26 实装 runtime
  readonly outputSchema?: unknown             // 预留，本版不实装
  readonly model?: string
}

export interface Handoff<TFrom = unknown, TTo = unknown> {
  readonly target: Agent<TTo, unknown>
  readonly kind: 'continuation' | 'as-tool'   // 两种语义
  readonly description?: string
  readonly inputFilter?: (history: Message[]) => Message[]
}

export interface Guardrail {
  readonly kind: 'input' | 'output' | 'tool'
  readonly name: string
  // 本版只占位 type；runtime 实装在 FEATURE_085 (v0.7.26)
}

export class Runner {
  static run<T>(agent: Agent, input: string | Message[], opts?: RunOptions): Promise<RunResult<T>>
  static runStream<T>(agent: Agent, input: string | Message[], opts?: RunOptions): AsyncIterable<RunEvent<T>>
}

export interface AgentReasoningProfile {
  default: ReasoningDepth       // 占位字段，FEATURE_078 v0.7.29 真实装
  max?: ReasoningDepth
  escalateOnRevise?: boolean
}
```

#### Handoff 两种语义

- **continuation**：上下文完整转移给目标 agent（对应 Scout → Generator/Planner 的角色升级）
- **as-tool**：目标 agent 变成一个 tool，被调时只拿生成的 input（对应 FEATURE_067 的 `dispatch_child_tasks`）

#### 内部 SA 路径 dog-food

**关键要求**：SA 直达路径必须改用新 Agent primitive 重写（AMA 路径在 FEATURE_084 v0.7.26 才重写）。理由：
- 如果 Agent primitive 只对外暴露、内部不用，会腐烂成装饰品
- SA 是最简单的路径，作为首个 dog-food 试点风险最小

现有 `runKodaX(options, prompt)` 入口保留作为 preset — 内部构造 "default coding agent" 实例，走 `Runner.run()`。外部调用者 zero breaking change。

## FEATURE_081: Compaction Layering and Session Base/Lineage Split

**Status**: Planned  
**Priority**: High  
**Category**: Core

### 背景

- `KodaXSessionLineage` 当前有 5 种 entry 类型，打包了血缘树 / compaction / artifact ledger / label / workspace
- 对 coding 场景这些是价值点，对外部 SDK 消费者过重
- FEATURE_072（v0.7.18）把 compaction 做到 lineage-native，但"通用 compaction 能力"从未被抽离
- FEATURE_011 的 `compactMessages()` 是 legacy，仍被导出（GLM F-9）

### 设计

#### Session 的分层

**基础 Session**（本版在 `packages/coding/src/primitives/session.ts`，v0.7.24 FEATURE_082 迁到 `@kodax/core`）：

```ts
export interface Session {
  readonly id: string
  append(entry: SessionEntry): Promise<void>
  entries(): AsyncIterable<SessionEntry>
  fork(opts?: { name?: string }): Promise<Session>
  readonly metadata: ReadonlyMap<string, unknown>
}

export interface SessionEntry {
  id: string
  ts: number
  type: string          // 扩展靠这个分派
  payload: unknown
}

export interface MessageEntry extends SessionEntry {
  type: 'message'
  payload: { role: MessageRole; content: ContentBlock[] }
}
```

**LineageExtension**（本版在 `packages/agent/src/extensions/lineage.ts`，v0.7.24 迁到 `@kodax/session-lineage`）：

```ts
export interface SessionExtension {
  readonly entryTypes: readonly string[]
  readonly operators: Record<string, (s: Session, ...args: any[]) => Promise<any>>
  readonly reducers?: Record<string, (entries: SessionEntry[]) => unknown>
}

export const LineageExtension: SessionExtension = {
  entryTypes: ['branch', 'rewind-marker', 'label', 'artifact-ledger', 'compaction'],
  operators: {
    branch: (s) => { /* KodaXSessionLineage.branch 逻辑 */ },
    rewind: (s, toId) => { /* ... */ },
    label: (s, name) => { /* ... */ },
    attachArtifact: (s, ref) => { /* ... */ },
  },
  reducers: {
    buildLineageTree: (entries) => { /* 从 entry 流重建血缘视图 */ },
  },
}
```

coding preset 默认组装 LineageExtension + CompactionExtension + ArtifactExtension，等价于今天的 `KodaXSessionLineage` 行为。

#### Compaction 的分层

```
Layer A:
  ├── CompactionPolicy 接口       —— shouldCompact / compact / restore
  ├── DefaultSummaryCompaction   —— 最简实现：token 阈值触发 + LLM summary 旧消息
  │                                 任何多轮 Agent 都开箱可用
  └── CompactionHook              —— Session 钩子

Layer B:
  └── LineageCompaction           —— 当前 FEATURE_072 能力保留到 @kodax/session-lineage

Layer C:
  └── coding preset pin LineageCompaction
```

```ts
export interface CompactionPolicy {
  name: string
  shouldCompact(session: Session, tokensUsed: number, budget: number): boolean
  compact(session: Session, ctx: CompactionContext): Promise<CompactionResult>
  restore?(session: Session, hint: unknown): Promise<void>
}
```

### Contract

**Public API（新增）**：Agent / Handoff / Guardrail / AgentReasoningProfile / Session / SessionEntry / MessageEntry / SessionExtension / CompactionPolicy / DefaultSummaryCompaction types + Runner class。临时从 `@kodax/coding` 导出，v0.7.24 迁到 `@kodax/core` / `@kodax/session-lineage`。

**Public API（保留）**：
- `runKodaX()` / `runManagedTask()` / `KodaXClient` 行为零变化
- `KodaXSessionLineage` 作为类型别名保留到 FEATURE_086（v0.7.27）清理前缀时移除
- `compactMessages()` 标 `@deprecated`，v0.7.27 移除

**Internal**：
- SA 路径用新 Runner + Agent 重写（dog-food）
- Scout/Generator/Evaluator 构造成 Agent 对象作为占位（但执行仍走旧路径，FEATURE_084 v0.7.26 才切运行时）

### Non-Goals

- **不**重写 AMA 路径（FEATURE_084 v0.7.26）
- **不**实装 Guardrail runtime（FEATURE_085 v0.7.26）
- **不**实装 reasoning profile 行为（FEATURE_078 v0.7.29）
- **不**改变 managed protocol（FEATURE_084 v0.7.26）
- **不**引入序列化机制（FEATURE_087 v0.7.28）
- **不**新增 post-compact reconstruction 能力（FEATURE_072 已做）
- **不**在本版进行包拆分（FEATURE_082 v0.7.24 才做）

### Dependencies

- **前置**：FEATURE_079 (v0.7.22) — task-engine 拆分后的代码基础
- **后置**：
  - FEATURE_082 (v0.7.24) — 包重组时把 A 层 types 迁到 `@kodax/core`
  - FEATURE_084 (v0.7.26) — 用 Agent primitive 重写 AMA 路径
  - FEATURE_060 (v0.7.30) — bounded-memory 在新 Session 模型上设计 eviction
  - FEATURE_078 (v0.7.29) — reasoning profile 填充 Agent.reasoning 字段实装
  - FEATURE_087+ 自构建 — 依赖 Agent / Tool 是 data

### 前置约束

- **FEATURE_076 约束**：`runManagedTask` 的 round-exit `context.messages` 语义必须与本版 Session base 兼容，v0.7.25 落地 076 时不受影响
- **FEATURE_060 约束**：bounded-memory 的核心 eviction 点必须在 Session base 上能插

### Testing

- 单元：Agent / Handoff / Guardrail 类型构造与 type-level 测试
- 单元：Runner.run 对最小 Agent + 单 tool 的 hello-world
- 单元：`Session` + `DefaultSummaryCompaction` 独立可用测试
- 单元：`Session` + `LineageExtension` 完整等价于当前 `KodaXSessionLineage` 行为（regression）
- 集成：SA 路径 end-to-end 对比 v0.7.22 behavior snapshot（必须一致）
- 集成：lineage compaction / branch / rewind 全流程（回归 FEATURE_019 / FEATURE_072 覆盖）
- SDK 消费者视角：`examples/embedded-agent.ts` 10 行构造 + 跑一个 agent

### Risk

- **Agent API shape 锁定**：v0.7.23 标 `@experimental`，v0.7.x 期间可能微调
- **Compaction hot path**：FEATURE_072 post-compact reconstruction 完整保留为 LineageCompaction 内部实现；新增 "long session regression" 套件跑 100+ 回合 fixture；compaction 实装做一轮 adversarial review

### Timeline

v0.7.23 合并两 feature。SA dog-food + Session 切分 + 全套回归即发版。
